# Project Roadmap

English | [简体中文](./ROADMAP.md)

## Current Version (v1.0.10)

- Basic disk information retrieval functionality
- Cross-platform support (Windows, macOS, Linux)
- TypeScript type definitions
- Node.js 12+ compatibility

## Short-term Plans (v1.1.0)

### New Features

- [ ] Retrieve information for all disks, not just the main disk
- [ ] Add disk usage percentage calculation
- [ ] Support custom disk path queries
- [ ] Add disk type identification (HDD/SSD/Network drive, etc.)

### Improvements

- [ ] Optimize error handling and error messages
- [ ] Enhance documentation and example code
- [ ] Add more platform support (FreeBSD, Android)
- [ ] Performance optimization

## Mid-term Plans (v1.2.0)

### New Features

- [ ] Disk performance monitoring (read/write speeds)
- [ ] Disk health status query
- [ ] File system information retrieval
- [ ] Disk partition information

### Improvements

- [ ] Add asynchronous API
- [ ] Streaming API support for large disks
- [ ] Caching mechanism to reduce system calls
- [ ] More unit tests and integration tests

## Long-term Plans (v2.0.0)

### New Features

- [ ] Complete disk management toolkit
- [ ] Disk space analysis and cleanup suggestions
- [ ] Disk activity history logging
- [ ] Disk performance benchmarking tools

### Architecture Improvements

- [ ] Plugin system for extensibility
- [ ] Microservices architecture support
- [ ] Web interface and visualization
- [ ] REST API service

## Contribution Priorities

1. **High Priority**
   - Retrieve information for all disks
   - Optimize error handling
   - Enhance documentation

2. **Medium Priority**
   - Disk performance monitoring
   - Asynchronous API
   - More platform support

3. **Low Priority**
   - Plugin system
   - Web interface
   - REST API service

## Release Schedule

- **v1.1.0**: Expected Q4 2025
- **v1.2.0**: Expected Q1 2026
- **v2.0.0**: Expected Q2 2026

_Note: The above plans may be adjusted based on community feedback and actual development progress_
